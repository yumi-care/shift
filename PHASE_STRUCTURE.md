# フェーズ構造 - 全体イメージ

---

## 6つのフェーズ全体像

```
Phase 1：法人・事業所・拠点の登録
  ↓
Phase 2：職員の登録
  ↓
Phase 3：職員がシフト希望を出す
  ↓
Phase 4：システムがシフト表を作成する
  ↓
Phase 5：勤怠システムから実績データを取得
  ↓
Phase 6：勤務体制表（実績）を自動で作成・印刷
```

---

## Phase 1：法人・事業所・拠点の登録

**何をする？**
- 法人（会社）を登録
- 事業所（施設）を登録
- 拠点（棟・ユニット）を登録

**どんな情報を登録する？**

### 法人の情報
```
法人名：ゆうみのいえ
法人番号：1234567890123
```

### 事業所の情報
```
事業所名：ゆうみのいえ（三本木）
サービス種別：共同生活援助
（このサービスには何人以上の職員が必要か、などが決まる）
```

### 拠点の情報
```
拠点名：三本木
所在地：愛知県豊橋市三本木町〇〇
営業時間：08:00 - 18:00
職員の人数枠：何人までシフトを作成できるか
```

**画面の流れ：**

```
【ログイン画面】
  ↓
【法人選択 or 新規作成】
  ユーザーが「ゆうみのいえ」を選択
  or
  新しく「新しい法人を作成」
  ↓
【事業所選択 or 新規作成】
  ユーザーが「ゆうみのいえ（三本木）」を選択
  or
  新しく「新しい事業所を作成」
  ↓
【拠点選択 or 新規作成】
  ユーザーが「三本木」を選択
  or
  新しく「新しい拠点を作成」
  ↓
【拠点が選択されました】
  ここから Phase 2 へ進む
```

---

## Phase 2：職員の登録

**何をする？**
- 働く人（職員）の情報を登録する

**どんな情報を登録する？**

```
職員の氏名：田中太郎
雇用形態：常勤 or 非常勤
資格：介護福祉士、ヘルパー 等
どこの拠点で働くか：三本木 or 複数拠点
週に何時間働くか：40時間 など
```

**画面の流れ：**

```
Phase 1 で選んだ拠点が、ここで使われる
  ↓
【職員を新規作成】
  ├─ 氏名を入力
  ├─ 雇用形態を選択（常勤 / 非常勤）
  ├─ 資格を選択
  ├─ 働く拠点を選択（この拠点 or 複数拠点）
  └─ 週の労働時間を入力
  ↓
【確認画面】
  修正 or 登録
  ↓
【職員が登録されました】
  ここから Phase 3 へ進む
```

---

## Phase 3：職員がシフト希望を出す

**何をする？**
- 各職員が「来月、この日は勤務できます」と入力する

**どんな情報を入力する？**

```
来月：2026年5月
この日に勤務できるか：
  5月1日（金）→ 勤務可能
  5月2日（土）→ 休み希望
  5月3日（日）→ 勤務可能
  ...

or

この時間帯で勤務できるか：
  08:00 - 17:00 → できる
  16:00 - 01:00 → できない
  など
```

**職員の使い方：**

```
職員 A がシステムにログイン
  ↓
【来月のシフト希望】
  カレンダーが表示される
  ↓
  「この日は勤務します」「この日は休みます」と入力
  ↓
【確認画面】
  修正 or 提出
  ↓
【提出完了】
```

---

## Phase 4：システムがシフト表を作成する

**何をする？**
- 職員たちの希望をもとに、自動でシフト表を作成する
- 作成されたシフト表をチェック・修正する

**どういう仕組み？**

```
管理者が事前に「ルール」を設定
  例）「月曜日 08:00-17:00 には、介護福祉士が2人必要」
  
職員たちが希望を出す
  
システムが自動で割り当てる
  「田中さんは資格があるし、この時間帯で勤務可能と書いてるから、月曜日に配置しよう」
  
管理者が確認
  「このシフト表でいいですか？」と画面に表示
  もし足りない時間帯があれば手動で調整
```

**画面の流れ：**

```
【シフト表（予定）の確認】
  ├─ カレンダー形式でシフト表が表示
  ├─ 足りない時間帯は赤く表示
  ├─ 十分な時間帯は緑で表示
  ├─ 修正が必要なら手動で変更
  └─ 「確定」ボタン
  ↓
【シフト表が確定しました】
```

---

## Phase 5：勤怠システムから実績データを取得

**何をする？**
- 実際に職員が勤務した時間を、勤怠システム（タッチオンタイム）から自動で取得する

**どういう仕組み？**

```
勤怠システム（タッチオンタイム）
  ↓（毎日、自動で）
このシステムに実績データを送信
  ├─ 誰が
  ├─ いつ
  ├─ 何時間働いたか
  ↓
【このシステムが記録】
  このデータは、後で「勤務体制表」を作るときに使う
```

**このシステムでやること：**
```
❌ 打刻（勤務時刻を記録する）
❌ 勤怠を編集（修正する）
✅ 勤怠データを受け取る（参照するだけ）
```

**例：**
```
田中さんが勤務システムで打刻する
  08:30 出勤
  17:15 退勤
  ↓
（勤怠システムから自動で）
このシステムが受け取る
  「田中さんが 2026/4/1 に 8時間45分 働いた」
```

## Phase 6：勤務体制表（実績）を自動で作成・印刷

**何をする？**
- Phase 4 で作ったシフト（予定）と Phase 5 で取得した実績を合わせて
- 「勤務体制表」というレポートを自動で作成する

**何に使う？**
- 行政に提出する書類ではない
- 施設内で保存しておくべき書類
- 「うちの施設は、この月、きちんと職員を配置していた」という記録

**どんな内容？**

```
【勤務体制表 2026年4月】

事業所名：ゆうみのいえ（三本木）
サービス種別：共同生活援助

【職員の勤務実績】
  田中太郎：
    4/1（月）08:30-17:15
    4/2（火）08:00-17:00
    4/3（水）休み
    ...
    
  鈴木花子：
    4/1（月）16:00-01:00
    ...

【集計】
  4月の平均職員数：5.2人
  常勤換算数：4.8人
  （この数字が、法律で決められた人数以上か確認）
```

**画面の流れ：**

```
【勤務体制表が自動で生成されました】
  ├─ 確認画面で内容をチェック
  ├─ PDF として印刷可能
  ├─ システムに保存
  └─ ダウンロード可能
```

## 全体の流れ（まとめ）

```
Phase 1：「うちの法人・事業所・拠点の情報を登録しよう」
  ↓
Phase 2：「働いている職員の情報を登録しよう」
  ↓
Phase 3：「職員に、来月のシフト希望を聞こう」
  ↓
Phase 4：「希望をもとに、シフト表を自動で作ろう」
  ↓
Phase 5：「実際に働いた時間を勤怠システムから取得しよう」
  ↓
Phase 6：「シフト表と実績から、勤務体制表を作ろう」
```
