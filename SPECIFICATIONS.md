## 1. プロダクト概要

| 項目 | 内容 |
|------|------|
| **プロダクト名** | 介護・障害福祉シフト自動化システム |
| **形態** | マルチテナント型 SaaS |
| **対象ユーザー** | 介護・障害福祉サービス事業者（法人〜大規模チェーン） |
| **主要対象地域** | 愛知県（豊橋市・東三河広域連合）→ 順次拡大 |
| **最終成果物** | Webアプリケーション（ブラウザベース） |

---

## 2. 設計思想

### コア思想
> **「最初の設定だけで、あとは完全自動」**
> - 管理者の工数をゼロに近づける
> - 勤怠システム（タッチオンタイム）との完全統合
> - 法務・法規制対応の自動化

### 実装の約束
- 管理者が日々やることは「初期設定」と「例外対応」のみ
- シフト作成や勤務体制表出力は**全自動**
- 人員基準・法的要件は自動チェック

---

## 3. 機能仕様（6フェーズ）

### Phase 1：事業所登録

**目的：** システムの基盤となる法人・事業所・拠点を登録

**データ階層**
```
法人（Corporation）
 └─ 事業所（Facility）
     └─ 拠点（Location）
```

**登録項目**

| レベル | 項目 | 説明 |
|--------|------|------|
| **法人** | 法人名 | 大分類 |
| | 法人番号 | 一意の ID |
| | 管理者アカウント | 法人全体の管理者 |
| **事業所** | 事業所名 | 施設名 |
| | 所在地 | 住所 |
| | 事業所番号 | 行政届け出番号 |
| | サービス種別 | 介護・障害福祉の種別 |
| | 人員基準 | 国が定める基準（自動適用） |
| | 定員数 | 利用者定員 |
| | 開所日・営業時間 | 運営スケジュール |
| **拠点** | 拠点名 | 支店名（例：三本木） |
| | 所在地 | 住所 |
| | 拠点設定 | 拠点独自のルール（必要な場合） |

**特徴**
- 拠点は**任意**（複数拠点なし事業所もOK）
- シフト・勤怠は**拠点単位**で管理
- 拠点データは**事業所単位で自動集計** → 勤務体制表生成

---

### Phase 2：スタッフ登録

**目的：** 各スタッフの属性・配置を登録し、シフト自動割当の基盤を作成

**登録項目**

| 項目 | 説明 | 備考 |
|------|------|------|
| 氏名 | 姓・名 | 漢字・カナ対応 |
| スタッフ ID | 一意の ID | 勤怠システムとの連携 ID |
| 雇用形態 | 常勤・非常勤・パート | セレクト |
| 保有資格 | 看護師・介護福祉士・ヘルパー等 | 複数選択可能 |
| 職種 | 上記資格に基づく職種 | 拠点・シフトで使用 |
| 配置パターン | 1拠点専任 or 複数拠点勤務 | シフト提出時の振る舞いで使用 |
| **所属拠点（1拠点専任の場合）** | 固定拠点 | 提出時に自動セット |
| 所定労働時間 | 週 40h など | 常勤換算計算用 |
| 契約時間数 | 週 40h など | 常勤判定用 |
| メールアドレス | ログイン用 | |

**特徴**
- スタッフの「配置パターン」によってシフト提出時の選択肢が変わる
  - **1拠点専任** → 拠点を自動セット（選択不要）
  - **複数拠点勤務** → 日ごとに拠点を自分で選択 or 管理者が振り分け

---

### Phase 3：シフト提出（スタッフから管理者へ）

**目的：** スタッフが翌月の希望シフト・希望休を提出

**提出形式**

| 項目 | 説明 |
|------|------|
| **対象月** | 提出対象の年月 |
| **シフト種別** | A（08:00〜17:00）、B（16:00〜01:00）等 |
| **拠点選択** | 1拠点専任スタッフは自動 / 複数勤務スタッフは日ごとに選択 |
| **希望休** | 休みたい日を指定 |
| **特記事項** | 事由がある場合は記入 |

**提出ルール**

- **スタッフが入力**（代理入力は行わない）
- **拠点選択**
  - 1拠点専任：固定（自動セット）
  - 複数拠点：日ごとに選択 or 管理者が曜日単位で振り分け可能

**提出期限管理**
- 管理者が「提出期限」を設定
- 未提出スタッフを自動表示

---

### Phase 4：シフト表（予定）作成・確認

**目的：** スタッフの希望をもとに、勤務予定を自動生成・確認・調整

**基本フロー**

```
1. 管理者がシフトテンプレートを事前登録
   └─ テンプレート例：「ゆうみのいえ三本木 / 08:00〜17:00（Aシフト） 
                         / 介護福祉士 2名・ヘルパー 1名」

2. スタッフがシフト希望を提出

3. システムがテンプレート × スタッフ資格で自動割当
   └─ 条件：「介護福祉士資格を持っており、Aシフト希望を出した」など

4. 管理者が画面で確認
   └─ 不足・重複がアラート表示
   └─ 人員基準の充足確認
   └─ 必要に応じて手動で調整

5. シフト確定
```

**画面表示**

- **月次カレンダー形式**：スタッフ × 日付のマトリクス
- **色分け表示**
  - 🔴 赤：人員不足（必要人数に満たない）
  - 🟡 黄：過剰配置（必要以上）
  - 🟢 緑：基準充足

**人員基準チェック**
- サービス種別ごとの「国の人員基準」を自動適用
- 例：介護老人福祉施設 → 3:1配置基準
- 不足箇所をアラート表示

**自動生成の制約**
- 希望休は必ず尊重
- 連続勤務の上限（例：夜勤は連続 3 日まで）
- 資格要件（この時間帯に配置できる職種は？）

---

### Phase 5：実勤怠データ連携

**目的：** 外部勤怠システム（タッチオンタイム）から実績データを自動取得

**設計原則**

> **「二重管理をしない」**
> - 勤怠管理（打刻・労務）は全て外部システムに委任
> - このツールは実勤怠データを「参照」するだけ

**役割分担**

| システム | 責務 |
|---------|------|
| **タッチオンタイム** | 打刻・実勤怠記録・労働基準法対応・給与計算 |
| **本ツール** | 勤務体制表（実績）生成のための参照取得 |

**連携方式**
- **API 自動同期**（手動 CSV インポート不要）
- タッチオンタイムのAPIから定期的に取得
- 取得サイクル：日次 or リアルタイム

**取得データ**

| データ | 説明 |
|--------|------|
| 出勤時刻 | 実際の開始時刻 |
| 退勤時刻 | 実際の終了時刻 |
| 欠勤区分 | 欠勤・有給・特別休暇など |
| 実労働時間 | 計算済みの勤務時間 |

**このツール内では記録・編集しない**
- 打刻修正 → タッチオンタイムで
- 有給申請 → タッチオンタイムで
- 給与計算 → タッチオンタイムで

**将来の拡張**
- 他社勤怠システム（ジョブカン等）への対応は API/Webhook 連携を基本とする

---

### Phase 6：勤務体制表（実績）自動生成

**目的：** 法規制で求められる「勤務体制表」を自動生成・保存・出力

**法的位置づけ**
- **作成義務** ：介護・障害福祉事業者に法律で課せられた義務
- **保存目的** ：事業所内での保管・監査対応
- **提出先** ：なし（行政提出ではない）

**生成ロジック**

```
Phase 4 の予定シフト（テンプレート × 希望）
    ↓ 融合 ↓
Phase 5 の実勤怠データ（タッチオンタイムから取得）
    ↓
勤務体制表（実績）自動生成
```

**出力フォーマット**

対象地域の様式に準拠（統一フォーマット）：

```
【ヘッダー】
- サービス種別（共同生活援助 など）
- 年月（2026年4月）
- 事業所名
- 記載期間（4週 or 暦月）
- 予定 / 実績の別
- 常勤が勤務すべき時間数（週・月）

【スタッフ一覧】（1行 1 スタッフ）
- No. / 職種 / 勤務形態（A/B/C/D） / 資格 / 氏名
- 日ごとの勤務時間（第1週〜第5週 × 7日）
- 勤務時間合計 / 週平均 / 兼務状況

【集計】
- サービス提供時間合計
- 平均利用者数
- 区分別延べ利用者数
- 開所日数

【人員基準チェック】
- 職種別：専従/兼務 × 常勤/非常勤 の実人数
- 常勤換算数（自動計算）
```

**勤務形態コード**

| コード | 区分 |
|--------|------|
| A | 常勤・専従 |
| B | 常勤・兼務 |
| C | 非常勤・専従 |
| D | 非常勤・兼務 |

**保存・出力**
- システム上に月次で自動保存
- PDF 印刷対応
- Excel ダウンロード対応

**拠点 → 事業所の集計**
- 複数拠点の実績 → 事業所単位で自動集計
- 拠点ごとの勤務体制表も同時生成

## 4. データモデル（概要）

| テーブル | 説明 | 主キー |
|---------|------|--------|
| corporations | 法人 | corp_id |
| facilities | 事業所 | facility_id |
| locations | 拠点 | location_id |
| staff | スタッフ | staff_id |
| qualifications | 資格マスタ | qualification_id |
| staff_qualifications | スタッフ × 資格 | staff_id + qualification_id |
| shift_templates | シフトテンプレート | template_id |
| shift_submissions | スタッフの希望提出 | submission_id |
| attendance_sync | タッチオンタイム連携 | sync_id |
| duty_roster | 勤務体制表 | roster_id |
