# システム導線（正確な仕様）

## 🔄 全体フロー

```
Phase 1（管理者）
  ↓ 法人・事業所・拠点登録
Phase 2（管理者）
  ↓ スタッフ登録：2種類に分岐
  ├─ 【固定勤務】：勤務曜日を登録
  └─ 【シフト申告制】：勤務曜日を空欄で登録
Phase 3（スタッフ）
  ↓ 申告方法が異なる
  ├─ 【固定勤務】：自動申告済み（勤務曜日から自動生成）
  └─ 【シフト申告制】：手動で日付・拠点を入力
ダッシュボード（管理者）
  ↓ 申告状況の表示
  ├─ 【固定勤務】：勤務曜日 + 「申告済み」
  └─ 【シフト申告制】：「-」+ Phase 3で手動入力した日数
```

---

## 📋 各フェーズの詳細

### **Phase 1：法人・事業所・拠点登録**
- **担当者**：管理者
- **登録内容**：
  - 法人名（例：ゆうみ株式会社）
  - 事業所名（例：ゆうみのいえ）
  - 拠点（例：三本木、江島、牛川）

---

### **Phase 2：スタッフ登録**
- **担当者**：管理者
- **登録対象**：全スタッフ
- **登録項目**：
  - スタッフ名
  - 職種
  - **勤務曜日**（重要：ここで分岐）
    - **有**：月火水木金（固定勤務）
    - **無**：空欄（シフト申告制）
  - 勤務時間

---

### **Phase 3：シフト申告**
- **担当者**：スタッフ（本人）
- **URL例**：`/phase3?corp_id=1&facility_id=1&year=2026&month=4`

#### **【固定勤務スタッフ】**
```
操作：スタッフを選択
↓
自動処理：該当月の勤務曜日をすべて「申告済み」に
↓
保存先：shift_submissions_1 { 1: { 2026-04-01, 2026-04-02, ... } }
↓
表示：「✓ 申告完了」
```

#### **【シフト申告制スタッフ】**
```
操作：
  1. スタッフを選択
  2. カレンダーから日付を選択
  3. 各日付の拠点を選択
  4. 送信
↓
保存先：shift_submissions_1 { 2: { 2026-04-05, 2026-04-10, ... } }
↓
表示：確認画面
```

---

### **ダッシュボード**
- **担当者**：管理者
- **表示対象**：**申告があったスタッフのみ**

#### **表示構成**
| スタッフ名 | 職種 | 勤務曜日 | 勤務時間 | 申告状況 |
|-----------|------|--------|--------|--------|
| 田中太郎（固定） | サ責 | 月火水木金 | 09:00～18:00 | 申告済み（22日） |
| 佐藤花子（シフト） | ヘルパー | - | - | 申告済み（3日） |

**重要：**
- 申告なしのスタッフは一切表示しない
- 「申告なし」という状態は、ダッシュボードに存在しない
- 申告があった人だけが表示される

---

## 📊 データ構造

### Phase 2（スタッフ登録データ）
```json
{
  "staffs_1": [
    {
      "staff_id": 1,
      "staff_name": "田中太郎",
      "position": "サービス提供責任者",
      "work_days": "月火水木金",  // ← 固定勤務
      "work_hours_start": "09:00",
      "work_hours_end": "18:00"
    },
    {
      "staff_id": 2,
      "staff_name": "佐藤花子",
      "position": "ヘルパー",
      "work_days": "",  // ← シフト申告制（空）
      "work_hours_start": "",
      "work_hours_end": ""
    }
  ]
}
```

### Phase 3（申告データ）
```json
{
  "shift_submissions_1": {
    "1": {
      "2026-04-01": { "staff_id": "1", "staff_name": "田中太郎", "location_name": "（勤務曜日により自動申告）" },
      "2026-04-02": { "staff_id": "1", "staff_name": "田中太郎", "location_name": "（勤務曜日により自動申告）" },
      ...
    },
    "2": {
      "2026-04-05": { "staff_id": "2", "staff_name": "佐藤花子", "location_name": "三本木" },
      "2026-04-10": { "staff_id": "2", "staff_name": "佐藤花子", "location_name": "江島" },
      ...
    }
  }
}
```

### ダッシュボード（表示ロジック）
```
shift_submissions_1 に存在するスタッフのみを表示
  ↓
各スタッフの申告数をカウント
  ↓
Phase 2から職種・勤務曜日・勤務時間を取得
  ↓
テーブルに表示
```

## ⚠️ 現在の問題点

1. **データソースの混在**
   - Phase 1：バックエンド API（http://localhost:8000）
   - Phase 2：localStorage
   - Phase 3：localStorage
   - ダッシュボード：localStorage + API

2. **データ同期の問題**
   - Phase 1 で登録した法人・事業所が Phase 2・3 で正しく参照されるか不明確

3. **「申告なし」の表現**
   - 現在の実装：申告がないスタッフを表示
   - 正しい実装：申告があったスタッフのみ表示
